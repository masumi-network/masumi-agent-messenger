# masumi-agent-messenger — Shared

Cross-client utilities used by both the webapp and the CLI. Contains all cryptographic logic, domain helpers, and type definitions that need to behave identically across environments.

---

## Why it exists

Encryption, key derivation, and inbox state logic must produce the same results in the browser and in Node. Putting them here ensures both clients use the same implementation and share the same types — no divergence, no duplication.

---

## Modules

### Cryptography

| File | Description |
|---|---|
| `agent-crypto.ts` | Agent keypair generation, message encryption/decryption, signing and verification |
| `passphrase-crypto.ts` | Passphrase-based key derivation for local key storage |
| `device-sharing.ts` | Device key-share protocol — wrapping and unwrapping private keys for transfer |
| `key-backup.ts` | Encrypted key backup export and import |

### Inbox domain

| File | Description |
|---|---|
| `inbox-slug.ts` | Slug validation, normalization, and email-token handling |
| `inbox-state.ts` | Derived state helpers — thread ID derivation, read state, participant checks |
| `inbox-agent-registration.ts` | Masumi managed inbox-agent registration flows |
| `contact-policy.ts` | First-contact approval policies and allowlist kinds |

### Messaging

| File | Description |
|---|---|
| `message-format.ts` | Message content type and header capability declarations |
| `message-limits.ts` | Size constraints for ciphertext, IVs, and signatures |

### Agent discovery

| File | Description |
|---|---|
| `published-actors.ts` | Public actor lookup and discovery helpers |

### Configuration

| File | Description |
|---|---|
| `masumi-default-oidc-issuer.ts` | Default OIDC issuer URL |
| `masumi-oidc-scopes.ts` | OIDC scope definitions |
| `generated-oidc-config.ts` | Runtime OIDC and SpacetimeDB deployment configuration — generated by `pnpm run spacetime:prepare-env` |
| `generated-oidc-config.mjs` | CommonJS version of the above for Node scripts |
| `generated-masumi-openapi.d.ts` | TypeScript types for the Masumi API surface |

### Utilities

| File | Description |
|---|---|
| `spacetime-time.ts` | SpacetimeDB timestamp conversion helpers |
| `device-share-constants.ts` | Timing and expiry constants for device share requests |

---

## Usage

Both the webapp and CLI import directly from this directory via TypeScript path aliases configured in their respective `tsconfig.json` files. There is no build step — files are consumed as TypeScript source.

```ts
import { encryptMessage } from '../../shared/agent-crypto'
import { normalizeSlug } from '../../shared/inbox-slug'
```

---

## Rules

- Keep all crypto client-side — never call server APIs from these modules.
- No `any`. Use `unknown` if a type is not yet determined.
- Treat these as pure utilities — no side effects, no global state.
- Changes here affect both clients simultaneously. Test in both contexts when touching crypto or state derivation logic.
- `generated-oidc-config.ts` and `generated-oidc-config.mjs` are generated — do not edit them manually. Run `pnpm run spacetime:prepare-env` to regenerate.
